# <a name="authoring-the-xml-schema-file-for-a-management-odata-web-service"></a>撰寫 Management OData Web 服務的 XML 結構描述檔案

定義資源之後，會公開您的 web 服務 (請參閱[撰寫管理 OData web 服務的 MOF 結構描述檔](./authoring-the-mof-schema-file-for-a-management-odata-web-service.md))，您將這些資源對應至實作支援的基礎 Windows PowerShell cmdlet藉由建立 XML 檔案符合每個資源的作業[資源對應的結構描述](./resource-mapping-schema.md)。 XML 檔案也會指定用戶端用來存取資源的 Url。

## <a name="mappng-resources-to-urls"></a>Mappng 資源 url

XML 檔案的第一個部分會對應至用來存取這些 Url 的 MOF 結構描述檔案中定義的資源。 下列範例會顯示該對應。

```xml
<?xml version="1.0" encoding="utf-8"?>
<ResourceMetadata xmlns="http://schemas.microsoft.com/powershell-web-services/2010/09">
    <SchemaNamespace>PswsTest</SchemaNamespace>
    <ContainerName>PSWSEntityContainer</ContainerName>
    <Resources>
        <Resource>
            <RelativeUrl>Service</RelativeUrl>
            <Class>PswsTest_Service</Class>
        </Resource>
        <Resource>
            <RelativeUrl>Process</RelativeUrl>
            <Class>PswsTest_Process</Class>
        </Resource>
    </Resources>
```

## <a name="mapping-cmdlets-to-crud-operations"></a>對應至 CRUD 作業的 cmdlet

然後在您指定之 cmdlet 的對應至 CRUD （建立、 讀取、 更新和刪除） 資源支援的作業。 在 管理 OData[資源對應的結構描述](./resource-mapping-schema.md)，對應的 CRUD 作業，如下所示。

|CRUD 命令|XML 項目|
|------------------|-----------------|
|建立|建立|
|讀取|查詢|
|更新|更新|
|刪除|刪除|
